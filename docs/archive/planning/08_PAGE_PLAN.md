# CustomerOps Agent｜页面规划

**版本**：v1.0
**日期**：2026-06-11

## 页面总览

MVP 共 3 个页面 + 1 个可选区域：

| # | 页面 | 路由 | 必做 | 说明 |
|---|------|------|------|------|
| 1 | 工单分析工作台 | `/` | ✅ | 首页，输入工单 + 启动分析 |
| 2 | 分析结果详情 | `/result/:ticketId` | ✅ | 展示完整 Agent 处理结果 |
| 3 | Eval / Bad Case | `/eval` | ✅ | 展示评估用例和结果 |
| 4 | About / Architecture | 首页底部区域 | ⬜ 可选 | 项目简介，不单独做页面 |

## Page 1：工单分析工作台（首页）

### 页面目标

用户输入一条售后工单，选择示例工单或手动输入，点击提交后启动 Agent 分析流程。

### 页面布局

```
┌─────────────────────────────────────────────────┐
│  顶部导航栏：项目名称 + 简介                       │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │  工单输入区                               │    │
│  │  ┌─────────────────────────────────┐    │    │
│  │  │  textarea: 售后问题描述（必填）    │    │    │
│  │  └─────────────────────────────────┘    │    │
│  │  ┌──────────────┐ ┌──────────────┐     │    │
│  │  │ order_id     │ │ product_type │     │    │
│  │  └──────────────┘ └──────────────┘     │    │
│  │  [ 提交分析 ]                            │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │  示例工单                                │    │
│  │  [ 物流延迟 ] [ 产品故障 ] [ 发票问题 ]   │    │
│  │  [ 退款申请 ] [ 投诉处理 ] [ 更多... ]    │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │  最近分析概览（如有）                      │    │
│  │  最近一次分析的摘要卡片                    │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 核心模块

| 模块 | 说明 |
|------|------|
| TicketInput | 工单输入表单（textarea + order_id + product_type + 提交按钮） |
| ExampleTicketList | 示例工单标签列表，点击自动填充 |
| RecentAnalysisCard | 最近一次分析的摘要卡片（可选，MVP 可简化） |

### 用户操作

| 操作 | 说明 |
|------|------|
| 输入工单 | 在 textarea 中输入售后问题描述 |
| 填写订单号 | 可选，在 order_id 输入框填写 |
| 选择产品类型 | 可选，下拉选择 earbuds / headphones / other |
| 选择示例 | 点击示例标签，自动填充输入框 |
| 提交分析 | 点击按钮，发送到后端，跳转结果页 |

### 对应 API

| API | 方法 | 路径 | 用途 |
|-----|------|------|------|
| Analyze API | POST | /api/analyze | 提交工单，触发 Agent 流程 |

### 状态设计

#### Loading 状态

- 提交按钮显示 spinner + "分析中..." 文字
- 按钮变为 disabled
- 输入框变为 disabled
- 不使用全屏 loading

#### Error 状态

- API 调用失败时，按钮下方显示红色错误提示
- 错误提示包含错误描述 + 重试按钮
- 输入框恢复可用

#### Empty 状态

- 初始状态：输入框为空，示例工单可见
- 最近分析区域：显示 "暂无分析记录" 灰色文字

#### Success 状态

- 提交成功后，跳转到 `/result/:ticketId`
- 或者在当前页下方展示 Agent Timeline + 结果（单页模式）

### 核心模块

| 模块 | 说明 | 优先级 |
|------|------|--------|
| AgentTimeline | 6 个 Agent 的执行状态时间线 | P1 |
| TicketOriginal | 工单原文展示 | P1 |
| FinalResultSummary | 最终处理建议摘要（status + risk） | P2 |
| ReplyCard | 客服回复草稿 + 复制按钮 | P3 |
| IntentResultCard | 意图识别结果 | P7 |

| PolicyResultCard | 政策判断结果 | P2 |
| QAResultCard | QA 质检结果 | P6 |
| RawJsonPanel | 完整 JSON（折叠） | P8 |

### 用户操作

| 操作 | 说明 |
|------|------|
| 查看 Timeline | 浏览 Agent 执行状态 |
| 复制回复 | 点击 "复制回复" 按钮 |
| 展开 Agent 详情 | 点击 Timeline 节点查看详情 |
| 展开 Raw JSON | 点击折叠面板查看完整 JSON |
| 返回首页 | 点击面包屑或导航返回 |

### 对应 API

| API | 方法 | 路径 | 用途 |
|-----|------|------|------|
| 分析结果 | GET | /api/result/:ticketId | 获取分析结果（或使用 POST /api/analyze 的响应） |

### 状态设计

#### Loading 状态

- 页面加载时，Timeline 节点逐个完成（动画）
- 如果是实时分析，每个 Agent 完成后对应节点变绿
- 结果卡片逐个出现（不是一次性全部显示）

#### Error 状态

- Agent 失败：Timeline 节点变红，显示错误描述
- 工具调用失败：ToolResultPanel 显示错误信息，红色标记
- API 失败：页面顶部显示错误提示 + 重试按钮

#### Empty 状态

- Evidence 为空：显示 "未检索到相关政策" + 建议补充信息
- Tool Results 为空：显示 "无工具调用结果"
- 工单号不存在：显示 "未找到该工单"

#### Success 状态

- 所有 Agent 完成，Timeline 全绿
- need_human_review = false 时，回复卡片绿色色条
- need_human_review = true 时，回复卡片黄色色条 + 人工审核提示

## Page 3：Eval / Bad Case 页面

### 页面目标

展示固定的评估用例，运行评估，展示通过 / 失败结果和原因。

### 页面布局

```
┌──────────────────────────────────────────────────┐
│  顶部导航栏                                        │
├──────────────────────────────────────────────────┤
│                                                  │
│  ┌──────────────────────────────────────────────┐│
│  │  Eval 控制区                                  ││
│  │  [ 运行全部评估 ]                              ││
│  │  通过：8 / 10  失败：2 / 10                    ││
│  └──────────────────────────────────────────────┘│
│                                                  │
│  ┌──────────────────────────────────────────────┐│
│  │  Eval 结果表格                                ││
│  │  ┌─────┬──────────┬──────────┬──────┬─────┐  ││
│  │  │ ID  │ 输入      │ Expected │Status│Issue│  ││
│  │  ├─────┼──────────┼──────────┼──────┼─────┤  ││
│  │  │ 01  │ 耳机故障  │ refund   │ Pass │ -   │  ││
│  │  │ 02  │ 缺订单号  │ need_info│ Fail │ ... │  ││
│  │  │ ... │ ...      │ ...      │ ...  │ ... │  ││
│  │  └─────┴──────────┴──────────┴──────┴─────┘  ││
│  └──────────────────────────────────────────────┘│
│                                                  │
│  ┌──────────────────────────────────────────────┐│
│  │  Case 详情（点击行后展开）                       ││
│  │  输入：我的耳机左耳没声音...                     ││
│  │  Expected Intent: product_defect               ││
│  │  Actual Intent: product_defect                 ││
│  │  Expected Policy: refund_eligible              ││
│  │  Actual Policy: refund_eligible                ││
│  │  Issues: 无                                    ││
│  │  Improvement: 无                               ││
│  └──────────────────────────────────────────────┘│
│                                                  │
└──────────────────────────────────────────────────┘
```

### 核心模块

| 模块 | 说明 |
|------|------|
| EvalControlBar | 评估控制区（运行按钮 + 统计） |
| EvalTable | 评估结果表格（ID / 输入 / Expected / Status / Issue） |
| CaseDetailPanel | Case 详情面板（Expected vs Actual） |

### 用户操作

| 操作 | 说明 |
|------|------|
| 运行评估 | 点击 "运行全部评估" 按钮 |
| 查看结果 | 浏览表格中的 Pass / Fail 状态 |
| 查看详情 | 点击行展开 case 详情 |
| 查看失败原因 | 查看 Issue 列和详情中的 Expected vs Actual |

### 对应 API

| API | 方法 | 路径 | 用途 |
|-----|------|------|------|
| Eval API | POST | /api/eval | 执行评估，返回结果 |

### 状态设计

#### Loading 状态

- 运行按钮显示 spinner + "评估中..."
- 表格显示 skeleton 占位行

#### Error 状态

- API 失败：表格上方显示红色错误提示
- 单个 case 失败：该行 Status 显示 "Error"，Issue 显示错误描述

#### Empty 状态

- 未运行评估：表格区域显示 "点击运行评估查看结果" 灰色提示

#### Success 状态

- 运行完成：显示统计（通过 X / 失败 Y）
- Pass 行：Status 列显示绿色 "Pass" 标签
- Fail 行：Status 列显示红色 "Fail" 标签，Issue 列显示问题描述

## Page 4（可选）：About / Architecture 简介

### 页面目标

简要说明项目定位和技术架构，帮助访问者快速理解项目价值。

### 实现方式

**不单独做页面**，放在首页底部或作为折叠区域。

### 内容

| 模块 | 说明 |
|------|------|
| 项目简介 | 一句话说明项目定位 |
| 架构图 | 简单的 Agent 流程图（文本或 SVG） |
| 技术栈 | 使用的技术列表 |

### MVP 是否必做

**否**。如果时间允许可以在首页底部加一个简介区域，不为此单独做页面或路由。
