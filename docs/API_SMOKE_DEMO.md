# API Smoke Demo

## 1. Scope

This document describes the CustomerOps Agent API smoke demo.

**Important limitations:**

- This API exposes the local customer service agent workflow.
- It does **not** call a real LLM. Answers are generated by rule-based templates.
- It does **not** call a real logistics API. The logistics tool is a mock implementation.
- It does **not** query a real order system. Order data is simulated.
- This is a demo API for testing and development purposes only.

## 2. Endpoint

```
POST /api/agent/chat
```

## 3. Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `user_query` | `str` | Yes | User's question. Must not be empty. |
| `order_id` | `str \| null` | No | Optional order ID from upstream context. |
| `conversation_history` | `list[str]` | No | Optional conversation history. Max 5 recent messages. |
| `llm_profile` | `str` | No | Model profile: `mock` (default), `deepseek`, or `doubao`. Frontend only sends this public name. |

### Example: Customs query (mock)

```json
{
  "user_query": "清关延迟怎么办？",
  "llm_profile": "mock"
}
```

### Example: Refund query

```json
{
  "user_query": "退款多久到账？"
}
```

### Example: Logistics query with order_id

```json
{
  "user_query": "我的订单123456到哪了？",
  "order_id": "123456"
}
```

### Example: Logistics query without order_id (triggers fallback)

```json
{
  "user_query": "我的快递到哪了？"
}
```

## 4. Response Body

| Field | Type | Description |
|-------|------|-------------|
| `answer` | `str` | Generated answer text. |
| `route` | `str` | Route taken: `logistics_tool`, `rag_knowledge_base`, or `fallback`. |
| `intent` | `str` | High-level intent: `logistics`, `aftersale`, `trace`, `other`. |
| `detail_intent` | `str` | Detailed intent: `customs`, `refund`, `logistics_status`, etc. |
| `citations` | `list[dict]` | Citations from retrieved knowledge base chunks. |
| `fallback_triggered` | `bool` | Whether fallback was triggered. |
| `fallback_reason` | `str \| null` | Reason for fallback (e.g., `missing_order_id`). |
| `confidence` | `str` | Overall confidence: `high`, `medium`, or `low`. |
| `retrieved_doc_ids` | `list[str]` | Document IDs retrieved from knowledge base. |
| `order_id` | `str \| null` | Order ID if extracted or provided. |
| `tool_used` | `str \| null` | Tool used (e.g., `mock_logistics_tool`). |
| `answer_source` | `str` | Answer source: `mock` (default), `real_llm`, or `real_llm_fallback_mock`. |
| `llm_profile` | `str \| null` | LLM profile used: `mock`, `deepseek`, or `doubao`. |
| `llm_provider` | `str \| null` | LLM provider used (e.g., `openai_compatible`), null if mock. |
| `llm_model` | `str \| null` | LLM model used, null if mock. |

## 7. Limitations

- **This is a demo API**, not a production system.
- **No real LLM by default**: Answers are generated by rule-based templates, not a language model. Real LLM is optional via env vars (see [LLM Adapter docs](docs/LLM_ADAPTER.md)).
- **No real logistics API**: The logistics tool returns mock data.
- **No real order system**: Order queries are simulated.
- **No production deployment**: This API runs locally only.
- **Answer quality**: Current metrics are based on M9.5 evaluation report. See `docs/EVAL_REPORT_M9_5.md` for details.
